# Plan Issue Skill

## Usage
```
/plan-issue <issue-number>
```

Example:
```
/plan-issue 1
```

## What This Does
Invokes the **Planner agent** to:
1. Read the PRD/spec from Specifier
2. Read codebase context from `docs/CLAUDE.md` (generated by Scaffolder)
3. Break the feature into technical tasks
4. Identify files to create/change
5. Plan architecture and edge cases
6. Save detailed plan to `docs/plans/<issue-number>.md`

## Output

**File:** `docs/plans/<issue-number>.md`

Contains:
- Summary of what will be built
- Files to create (with purpose)
- Files to change (with specific edits)
- Architecture decisions
- Implementation approach
- Edge cases and risks
- Acceptance criteria checklist (from PRD)
- Questions and assumptions

## Pre-requisites
- Feature has been specified (`/specify-feature` run first)
- `docs/PRD.md` and `docs/spec/<issue>.md` exist
- Codebase has been scaffolded (`/scaffold-project` run first)

## When to Use This Skill

**Use it when:**
- PRD has been reviewed and approved
- Ready to start detailed technical planning
- Team agrees on the feature's scope and acceptance criteria

## Workflow After Planning

```
1. /specify-feature "..." → Specifier creates PRD
2. Team reviews and approves PRD
3. /plan-issue <number> → Planner creates technical plan in docs/plans/
4. Team reviews plan
5. /build-issue <number> → Builder implements
```

## Instructions for Claude

Extract the issue number from the user's input (e.g., `/plan-issue 42` → number is `42`).

Invoke the `planner` sub-agent with that issue number.

The planner will:
1. Read the PRD from `docs/PRD.md`
2. Read the spec from `docs/spec/<issue>.md`
3. Read the codebase context from `docs/CLAUDE.md`
4. Analyze relevant files with the Read tool
5. Create a detailed technical plan
6. Save to `docs/plans/<issue>.md`
7. Report completion

## For Team Session

**Teaching point:** "Planner reads the PRD (product spec) and translates it into technical tasks. This is where 'what' becomes 'how'."
