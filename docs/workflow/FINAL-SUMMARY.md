## Key Concepts Explained

### 1. Prompts vs Skills vs Hooks

**Prompts** — One-off thoughts
```
"Analyze this error"
"Should we refactor X?"
```

**Skills** — Repeatable workflows
```
/scaffold-project <repo>
/specify-feature "..."
/plan-issue <number>
/build-issue <number>
/generate-tests <file>
```

**Hooks** — Passive automation
```
Stop hook: Auto-creates draft PR
PostToolUse hook: Auto-lints files
```

### 2. Product vs Technical Planning

**Specifier (Product):**
- Asks: "What does the user need?"
- Produces: PRD with acceptance criteria, user stories, design
- Focus: User perspective, what "done" looks like

**Planner (Technical):**
- Asks: "How do we build this?"
- Produces: Technical plan with files to change, architecture, edge cases
- Focus: Implementation approach

**Separation prevents:** Debates during coding. Both parties align upfront.

### 3. Durable Documentation

All output lives in `docs/` (git-tracked):
- NOT in chat history (which disappears)
- Searchable with grep
- Versioned with git
- Team can reference for months/years

### 4. Templates Ensure Consistency

**PRD template** → All PRDs have same sections
**ADR template** → All decisions documented same way

Agents read templates and fill them in.

### 5. Parallelization for Speed

Large features (8+ files) use sub-agents + worktrees:
- Sub-agent 1: Database (worktree-db)
- Sub-agent 2: API (worktree-api)
- Sub-agent 3: Frontend (worktree-ui)

Result: 2.5x speedup vs sequential
