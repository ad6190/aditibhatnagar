# Aditi's Blog — Codebase Guide

## Overview
A personal blog site built with React 19, Vite, and TypeScript. Content is authored in an Obsidian vault (`blog-site/aditi-blog/`) and transformed into JSON/Markdown by a build script, then rendered by a React SPA. Deployed to GitHub Pages via GitHub Actions on every push to `main`.

## Tech Stack
- **Frontend:** React 19, React Router v7, Vite 8.x, TypeScript 5.x, Tailwind CSS v4
- **Content:** Markdown (from Obsidian vault) + gray-matter frontmatter parsing
- **Comments:** Giscus (GitHub Discussions integration)
- **Animation:** framer-motion
- **Markdown rendering:** react-markdown + remark-gfm
- **Linting:** ESLint
- **Testing:** No tests yet (target: Vitest + @testing-library/react)
- **Deploy:** GitHub Actions (`.github/workflows/deploy.yml`)

## Build & Deploy Flow

### Content Pipeline
1. Author writes `.md` files in `blog-site/aditi-blog/` (Obsidian vault)
2. `npm run build` in `blog-site/site/` triggers `scripts/build-content.mjs`
3. Script parses frontmatter (title, date, tags) from all `.md` files
4. Generates `public/content/manifest.json` (list of posts) and individual markdown files
5. React app loads manifest at runtime, fetches content on-demand

### Build Process
```bash
cd blog-site/site/
npm run build    # Vite builds React + copies public/content/ to dist/
npm run lint     # ESLint check
npm run dev      # Local dev server
npm run test     # Vitest (once added)
```

### Deployment
- Push to `main` → GitHub Actions (`.github/workflows/deploy.yml`)
- Runs `npm run build` in `blog-site/site/`
- Deploys to GitHub Pages (`gh-pages` branch)
- Live at: https://aditibhatnagar.com (or GitHub Pages URL)

## Key Components & Files

### Navigation (Dark Mode Entry Point)
**File:** `blog-site/site/src/components/Navigation.tsx`

Current: Light-mode-only navbar with links to Home, About, Bookmarks, etc.

To add dark mode here:
- Create a theme context in `src/lib/theme.ts` or state in App.tsx
- Add toggle button in Navigation (sun/moon icon)
- Apply `dark:` Tailwind classes to components
- Persist choice to localStorage
- Update Giscus theme in `components/CommentSection.tsx` when theme changes

### Content Loading
**File:** `blog-site/site/src/lib/content.ts`

Loads `public/content/manifest.json` at app init. Safe to refactor; no external dependencies on its internals.

### Markdown Rendering
**File:** `blog-site/site/src/components/MarkdownRenderer.tsx`

Wraps `react-markdown` with remark plugins. Add Tailwind prose classes here if styling content.

## Common Tasks

### Adding a New Page
1. Create `.md` file in `blog-site/aditi-blog/`
2. Add frontmatter: `---\ntitle: ...\ndate: ...\ntags: [...]\n---`
3. Run `npm run build` to regenerate manifest
4. Create `.tsx` page component in `src/pages/`
5. Add route in `src/App.tsx`
6. Add nav link in `Navigation.tsx`

### Styling
- Tailwind v4: `src/index.css` (global), component files (local)
- No CSS Modules; use Tailwind utilities + inline styles
- Dark mode: use `dark:` prefix for dark-mode-specific classes
- Currently light-mode only (theme toggle not implemented)

### Testing Strategy
- Unit tests for utils/helpers: `src/__tests__/utils.test.ts`
- Component tests: `src/__tests__/Navigation.test.tsx` (once Vitest is set up)
- Integration: test content loading from manifest

## Known Gotchas
- **Visitor counter removed:** Feature was added (commit `db2fd54`) then removed (`f659db1`). Don't re-add without tests.
- **Giscus setup:** Requires repo ID and discussion category ID in env. See `src/components/CommentSection.tsx`.
- **Node version:** Tested on Node 18+. ESM build script requires `--experimental-modules` on older versions.
- **Tailwind v4:** Uses new CSS nesting syntax. Don't assume v3 class behavior.
- **React 19 Strict Mode:** Double-renders components in dev; this is expected (not a bug).
